# 1. Термины и определения

:::terms
| Термин | Определение |
|--------|-------------|
| URL | Uniform Resource Locator |
| Валидация | Проверка корректности данных |
:::

# 2. Исходные данные

Тестирование различных значений taskUrl для проверки корректной обработки гиперссылок.

# 3. Описание изменений

:::changes-table
| Описание функции «Как есть» | Описание функции «Как будет» |
|-----------------------------|------------------------------|
| Любое значение taskUrl создавало гиперссылку | Только валидные URL создают гиперссылку |
:::

# 4. Описание требований

## 4.1 Функция с валидным https URL

:::function-table{#func-valid-https}
function: Функция с валидным https URL
task: TASK-001
taskUrl: https://jira.example.com/browse/TASK-001
scenario: |
  **Ожидаемый результат:**
  - Должна создаться гиперссылка синего цвета с подчеркиванием

  **Основной сценарий:**
  1. Пользователь открывает документ
  2. Видит синюю ссылку TASK-001
  3. Может кликнуть на нее
:::

## 4.3 Функция с невалидным значением "—"

:::function-table{#func-dash}
function: Функция с невалидным значением "—"
task: TASK-003
taskUrl: —
scenario: |
  **Ожидаемый результат:**
  - НЕ должна создаваться гиперссылка
  - Текст TASK-003 должен отображаться обычным черным текстом без подчеркивания

  **Основной сценарий:**
  1. Пользователь открывает документ
  2. Видит обычный текст TASK-003
  3. Не может кликнуть на него
:::
