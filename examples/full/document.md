# 2. Исходные данные

В рамках данного ЧТЗ описываются требования к модификации интерфейса подписания документов в системе ЮЗЭДО.

Основные задачи:
- Улучшение пользовательского интерфейса
- Добавление поддержки массового подписания
- Интеграция с внешними сервисами проверки ЭП

# 3. Описание изменений

## 3.1 Общее описание

Планируется внедрение нового интерфейса подписания документов, который позволит:

1. Подписывать документы в один клик
2. Просматривать статус подписания в реальном времени
3. Получать уведомления о необходимости подписания

## 3.2 Описание функций «Как есть» / «Как будет»

:::changes-table
| Описание функции «Как есть» | Описание функции «Как будет» |
|-----------------------------|------------------------------|
| Подписание документов выполняется по одному | Реализовано массовое подписание до 100 документов |
| Отсутствует индикация прогресса | Добавлен прогресс-бар подписания |
| Нет уведомлений | Push-уведомления о новых документах |
:::

# 4. Описание требований

## 4.1 Функциональные требования

### 4.1.1 Массовое подписание документов

:::function-table{#func-1}
function: Массовое подписание документов в интерфейсе ЮЗЭДО
task: TASK-1001
taskUrl: https://task.example.com/browse/TASK-1001
scenario: |
  **Предусловия:**
  - Пользователь авторизован в системе
  - У пользователя есть права на подписание
  - Установлен криптопровайдер

  **Основной сценарий:**
  1. Пользователь переходит в раздел «Документы на подписание»
  2. Выбирает несколько документов чекбоксами
  3. Нажимает кнопку «Подписать выбранные»
  4. Система отображает диалог подтверждения
  5. Пользователь подтверждает действие
  6. Система последовательно подписывает документы
  7. Отображается прогресс-бар с текущим статусом

  **Ограничения:**
  - Максимум 100 документов за раз
  - Документы должны быть одного типа
:::

### 4.1.2 Уведомления о документах

:::function-table{#func-2}

scenario: |
  **Основной сценарий:**
  1. В систему поступает новый документ
  2. Система определяет ответственного подписанта
  3. Формируется push-уведомление
  4. Уведомление отправляется на устройство пользователя
  5. При клике на уведомление открывается документ
:::
